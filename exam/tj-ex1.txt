1. 요구사항 확인하기
[문항1]  도서 대여점 관리 프로그램 웹사이트를 개발 하려고 한다. 업무 분석으로 수집한 
정보를 이용하여 요구 사항을 분석 해야 한다. 도서 대여점 관리 프로그램에서는 무엇을 요구하는지 예를 들어 요구사항 분석내용에는 어
떤 것들이 있는 지 분석한 내용을 서술 하시오.
----
2개 이상의 분석 한 요구사항을 입력 하였을경우 [100점]
1개의 분석한 요구사항을 입력 하였을 경우 [60점]
입력 하지 못한 경우[0점]

(관리자)
도서 정보 관리 :
책 보유 목록 : 책 정보 출력, 정렬(입고일,이름,대여여부)기능, 검색기능, 총 갯수 표시, 대여된 책을 제외한 갯수 표시. 책 등록, 책 삭제
새로운 책 등록 -> 책 정보(입고일, 책이름, 권 수) 작성 -> 책 보유 목록에 책 정보 등록
버리는 책 ->  책 보유 목록에서 책 정보 삭제

도서 대여 :
1. 회원 채크 : 회원 목록에서 회원 채크(QR,바코드,이름,전화번호 등으로) 후 도서 대여 가능. 없을 시. 회원 등록 후 대여
2. 도서 대여 : 회원 정보 입력 -> 대여 정보(대여자, 대여일자, 대여일수, 대여권수, 대여한책) 작성 -> 대여 목록에 회원정보, 대여정보 등록 -> 대여된 책 목록에 등록(대여책 정보 관리)
3. 도서 반납 : 회원 정보 입력 -> 대여 정보 검색 -> 대여 정보 삭제 ->  대여 목록에서 삭제, 대여된 책 목록에서 삭제
대여 목록 : 입력된 정보를 토대로 목록 작성, 대여 정보(대여자,일수,남은기한,대여한책,대여권수) 표시, 정렬기능, 검색기능, 대여기간 지난 사람표시, 메시지 보내기
대여된 책 목록 : 입력된 정보를 토대로 목록 작성, 대여된 책 표시, 대여자 정보보기 가능, 정렬기능, 검색기능

회원 관리 :
1. 회원 목록 : 회원 생성시 입력한 정보 출력, 정렬기능, 대여정보(책, 기한 등) 검색기능, 이름 검색, 회원 등록 기능, 회원 삭제 기능, 대여기간 지난 회원 표시
2. 회원 등록 : 이름 , 전화번호, 주소? 주민동록번호? 등을 작성 후 서버에 등록 -> QR, 바코드, 카드 등 생성 -> 회원 전용 앱 또는 카드, 출력물 이용하여 회원에게 지급
3. 회원 삭제 : 회원 목록에 등록된 회원 정보 삭제. 데이터 남겨두지 않고 완전 삭제.

도서 주문 :
프로그램에 연계된 사업장 홈페이지 표시, 홈페이지에서 주문 등등 가능
책 보유 목록 사이드 메뉴로 띄움(기능 이용가능)

(매장 내 사용자 전용 컴퓨터)
도서 검색 : 책 보유 목록에서 도서 검색 -> 도서 여부, 도서 위치 표시, 대여 가능 여부 표시
직원 호출 : 직원에게 알람 -> 직원이 위치 확인 후 사용자 위치에 도착
문의하기 : 메시지 보내기, 직원 호출하기 가능
홈페이지 안내 : 사이트 주소(QR코드) 표시
