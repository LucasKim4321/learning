1. 요구사항 확인하기
[문항1]  도서 대여점 관리 프로그램 웹사이트를 개발 하려고 한다. 업무 분석으로 수집한 
정보를 이용하여 요구 사항을 분석 해야 한다. 도서 대여점 관리 프로그램에서는 무엇을 요구하는지 예를 들어 요구사항 분석내용에는 어
떤 것들이 있는 지 분석한 내용을 서술 하시오.
----
2개 이상의 분석 한 요구사항을 입력 하였을경우 [100점]
1개의 분석한 요구사항을 입력 하였을 경우 [60점]
입력 하지 못한 경우[0점]


도서 정보 관리 :
책 보유 현황 : 기존 책 목록 + 새로 들여온 책 - 버리는 책 
책을 새로 들여옴 -> 책 정보 작성 -> 책 현황에 등록
버리는 책 -> 책 정보 삭제 -> 책 현황에서 삭제
대여한 책을 제외한 보유 목록 : 책 보유 현황 - 책 대여 목록

도서 대여 정보 관리 : 
비회원 -> 회원 등록 후 대여
회원 -> 책을 빌림 -> 대여 정보 작성 -> 대여자 목록에 등록, 대여한 책 대여 목록에 등록
     -> 책을 반납 -> 대여 정보 삭제 ->  대여자 목록에 삭제, 대여한 책 대여 목록에 삭제

회원 정보 : 이름 , 전화번호, 주소?
